# module-calc 산식 명세 (영역 A~D)

> 사전원가 시뮬레이션의 핵심 계산 로직. 순수 Java(BigDecimal)로 구현되며 DB/Spring 의존성이 없다.
> 모든 반올림은 `RoundingMode.HALF_UP`.

## 영역 A — 전월 실적 (SAP BW 자동 적재)
| 기호 | 의미 | 소스 |
|------|------|------|
| K | 전월 사용량 | BW.실제배부수량 |
| L | 전월 원단위(kg/Ton) | BW.실제원단위_KGT |
| M | 전월 단가(원/kg) | BW.실제단가 |
| N | 전월 재료비(원) | BW.재료비 |
| O | 전월 톤당비용 | N / 총생산수량 × 1,000,000 |

## 영역 B — 당월 계획 (사용자 입력)
| 기호 | 의미 |
|------|------|
| X3 | 당월 생산량(톤) |
| X7 | 당월 폐품율(%) |
| Q | 당월 목표 원단위(기본값 = L) |
| R | 당월 입고수량(톤) |
| S | 당월 입고단가(원/kg) |
| T | 기초재고(톤) |
| U | 기초재고 단가(원/kg) |

## 영역 C — 당월 시뮬레이션 (자동 계산)
| 기호 | 산식 | 클래스 |
|------|------|--------|
| V | `(R*1000*S + T*1000*U) / (R*1000 + T*1000)` | `WeightedAvgPrice` |
| P | `Q × 분모톤(material, plan)` | `MonthlyUsage` |
| W | `P × V / 1,000,000` (백만원) | `MaterialCost` |
| X | `W / X3 × 1,000,000` (원/톤) | `CostPerTon` |

## 영역 D — 변동분석 (자동 계산)
| 기호 | 산식 | 클래스 |
|------|------|--------|
| Y | `(L - Q) × M × X3` 사용량차이 | `VolumeVariance` |
| Z | `Q × (M - V) × X3` 단가차이 | `PriceVariance` |
| AA | `Y + Z` 재료비종합 (양수=개선) | `TotalVariance` |

## 분모톤 결정 규칙 (`DenominatorResolver`)
| 자재 | 분모 |
|------|------|
| 펄프 (11xxxxx) | X3 전체 |
| 일반 고지 (12xxxxx) | X3 − 지관 − UKP |
| UKP / HINTON UKP (1100005, 1100023) | KB 톤 |
| ACB류 (13xxxxx) | ACB 톤 |
| 그 외 | X3 전체(기본값) |

> ⚠️ UKP/HINTON UKP 코드는 `11`로 시작하지만 펄프 규칙보다 **우선** 적용된다(KB 톤 사용).

## ✅ 핵심 검증 케이스 — PM2 / SC / 220 / LATEX(금호 KSL 252)

입력: `L=5.3643, M=1512.90, X3=16000, Q=5.20, V=1550`

| 항목 | 산식 결과(수학적 정확값) | 요청서 문서값 | 비고 |
|------|--------------------------|---------------|------|
| Y | **3,977,112** (= 3,977,111.52) | 3,977,498 | ⚠️ 약 386원 차이 |
| Z | **−3,086,720** | −3,086,720 | ✅ 일치 |
| AA | **890,392** (= 890,391.52) | 890,778 | ⚠️ Y 차이에서 기인 |

### 차이 원인 분석
- Z·V·Q·X3는 문서와 정확히 일치 → 산식 로직은 정상.
- Y가 문서값(3,977,498)이 되려면 **M ≈ 1513.047** 이어야 함.
- 즉, 문서에 표기된 `M=1512.90`이 실데이터(`usage_2604.xlsx`)의 실제 소수점 정밀도와 미세하게 다른 것으로 추정됨.
- **조치**: 산식 클래스는 수학적 정합성을 유지하고, JUnit 테스트는 정확값으로 검증. 실데이터 엑셀 확보 후 M의 정밀도를 확인하여 기대값을 최종 확정한다(`VarianceFormulaTest`의 `TODO[실데이터 확정]` 참조).

## 경계값 처리
- 가중평균 V: 분모(총 kg)가 0이면 V=0. R=0→V=U, T=0→V=S.
- 톤당비용 X: X3=0이면 0.
- 모든 산식: null 입력은 0으로 처리(NPE 방지).
- 분모톤: 자재코드 공백/null 또는 input null → `CalcException`.
